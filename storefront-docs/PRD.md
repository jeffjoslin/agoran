## 1. Vision & Purpose

### 1.1 The Problem We're Solving
SkillForge can autonomously discover trends, generate quality digital products, and is ready to publish — but it currently depends on third-party platforms (Gumroad, Lemon Squeezy, Ko-fi) that:
- Take 5–10% of every sale
- Require browser automation hacks instead of clean API calls
- Own the customer relationship and email data
- Can ban, restrict, or change policies at any time
- Were not built for agent-driven, high-volume, multi-sector publishing

### 1.2 The Vision
Build the **first agent-native digital product storefront** — a platform designed from the ground up to be published to by autonomous AI agents, at scale, across unlimited sectors and target audiences. 

The storefront is the **commercial face of the agentic collective**. Every product published here was discovered by an agent, built by an agent, and delivered by the platform automatically. As the collective grows to tens of thousands of agents, this platform scales with it.

### 1.3 Goals (MVP)
1. Replace third-party storefronts with a platform we own and control
2. Expose a clean REST API that SkillForge calls to publish products
3. Enable human buyers to discover, purchase, and receive digital products
4. Capture every buyer's email for list building
5. Provide Jeff with a dashboard to monitor sales and pipeline activity

### 1.4 Goals (Future — Post-MVP)
- Multi-brand sector storefronts (one engine, many faces)
- Agent attribution and royalty tracking
- On-chain product provenance (blockchain layer)
- Dynamic pricing intelligence
- Cross-sector recommendation engine
- Affiliate/referral system for agents and humans

## 2. Users & Personas

### 2.1 Buyer (Human)
- Arrives via SEO, social, or email
- Browses by sector/topic or searches for a specific need
- Purchases a digital product (PDF guide, checklist, template)
- Receives instant download + email confirmation
- May return for more products in same or adjacent sectors

**Key needs:** Trust, easy discovery, frictionless checkout, instant delivery

### 2.2 Agent Publisher (SkillForge / AI System)
- Authenticates via API key
- Calls API to create a product record
- Uploads the product file (PDF)
- Triggers publish — product goes live with auto-generated landing page
- Retrieves analytics on published products

**Key needs:** Clean API, clear error messages, predictable behavior, no rate limits that block overnight runs

### 2.3 Admin (Jeff)
- Views all products across all sectors
- Monitors revenue, conversion rates, top performers
- Can manually edit, unpublish, or feature a product
- Reviews pipeline health (how many products published, by which run)

**Key needs:** Single dashboard, actionable data, override controls

## 3. MVP Scope

### 3.1 In Scope

#### 3.1.1 Product Catalog & Discovery
- [ ] Homepage with featured products + sector navigation
- [ ] Sector pages (Finance, Health, Tech, Marketing, AI, etc.) — dynamically created when first product in a sector is published
- [ ] Product listing page with filtering (sector, type, price range, audience)
- [ ] Individual product landing page — auto-generated from product metadata
- [ ] Search (basic keyword search across title + description)

#### 3.1.2 Product Landing Page (Auto-Generated)
Each product page includes:
- [ ] Hero headline + subheadline (generated by LLM at publish time)
- [ ] "What you'll get" bullets (generated from product content)
- [ ] Who it's for (target audience)
- [ ] Price + Buy button
- [ ] Trust signals (secure checkout badge, instant download, money-back guarantee)
- [ ] Related products (same sector)

#### 3.1.3 Checkout & Payments
- [ ] Stripe Checkout (hosted, not custom — faster to ship, PCI handled)
- [ ] One-time payment only (subscriptions post-MVP)
- [ ] Email capture mandatory at checkout
- [ ] Coupon/discount code support (Stripe native)
- [ ] Order confirmation page

#### 3.1.4 Digital Delivery
- [ ] Product file stored in Cloudflare R2 (private bucket)
- [ ] On purchase: generate signed download URL (24-hour expiry)
- [ ] Deliver URL via email confirmation (Resend)
- [ ] Download page accessible via signed link (no account required for MVP)

#### 3.1.5 Agent Publishing API
Full spec in Section 5. Summary:
- [ ] `POST /api/v1/products` — create product
- [ ] `POST /api/v1/products/:id/assets` — upload file
- [ ] `PUT /api/v1/products/:id` — update metadata
- [ ] `POST /api/v1/products/:id/publish` — make live
- [ ] `GET /api/v1/products/:id` — retrieve product details
- [ ] `GET /api/v1/products/:id/analytics` — basic metrics
- [ ] `GET /api/v1/health` — pipeline health check

#### 3.1.6 Admin Dashboard
- [ ] Product list (all products, any status: draft/live/unpublished)
- [ ] Per-product stats: views, purchases, revenue, conversion rate
- [ ] Manual edit: title, description, price, sector, status
- [ ] Publish/unpublish controls
- [ ] Revenue summary (daily, weekly, all-time)
- [ ] Pipeline run log (which SkillForge run published what)

#### 3.1.7 Email (Transactional)
- [ ] Order confirmation with download link
- [ ] Welcome email for new buyers (first purchase)
- [ ] All via Resend (transactional) with Zoho as backup

### 3.2 Out of Scope (MVP)
- User accounts / login for buyers
- Subscriptions or recurring payments
- Reviews or ratings
- Affiliate system
- Multi-brand/sub-domain storefronts
- Blockchain/on-chain provenance
- Agent royalty attribution
- Multi-currency
- Refund automation (manual for now)
- Course/video content (PDF/file downloads only)
- Public API access (agent API is private/key-gated only)

## 4. User Stories

### Buyer
- As a buyer, I can browse products by sector so I can find what's relevant to me
- As a buyer, I can read a clear product page that tells me what I'm getting before I buy
- As a buyer, I can purchase with a credit card in under 60 seconds
- As a buyer, I receive a download link instantly via email after purchase
- As a buyer, I can re-download my product from the link within 24 hours

### Agent Publisher
- As an agent, I can create a new product via API with a single POST call
- As an agent, I can upload a PDF file to an existing product record
- As an agent, I can update product metadata (title, description, price) before publishing
- As an agent, I can publish a product and have it immediately live on the storefront
- As an agent, I can retrieve analytics for any product I published
- As an agent, I receive clear error codes when something goes wrong so I can log and retry

### Admin
- As an admin, I can see all products and their live/draft status at a glance
- As an admin, I can see today's revenue and which products drove it
- As an admin, I can manually edit any product's details
- As an admin, I can unpublish a product without deleting it
- As an admin, I can see which SkillForge pipeline run published each product

## 5. Agent API — Full Specification

### 5.1 Authentication
- API Key authentication via `Authorization: Bearer <key>` header
- Keys stored in Supabase, hashed, with `agent_id` attribution
- Rate limit: 100 requests/minute per key (generous for overnight batch runs)
- All API endpoints return JSON

### 5.2 Endpoints

#### `GET /api/v1/health`
Pipeline connectivity check. No auth required.
```json
Response 200:
{ "status": "ok", "timestamp": "2026-03-08T00:00:00Z" }
```

---

#### `POST /api/v1/products`
Create a new product in draft state.

**Request:**
```json
{
  "title": "The Solo AI Stack: 47 Tools to Replace Your Entire Team",
  "description": "A comprehensive guide to...",
  "short_description": "47 AI tools. One person. Full team output.",
  "sector": "ai",
  "sub_niche": "productivity",
  "product_type": "pdf_guide",
  "target_audience": ["solopreneurs", "freelancers", "small business owners"],
  "price_cents": 1700,
  "currency": "usd",
  "tags": ["AI tools", "productivity", "automation"],
  "agent_id": "skillforge-run-2026-03-08",
  "run_metadata": {
    "trend_score": 7.4,
    "quality_score": 8.1,
    "pipeline_run_id": "run_abc123"
  }
}
```

**Response 201:**
```json
{
  "product_id": "prod_xyz789",
  "status": "draft",
  "created_at": "2026-03-08T02:14:00Z",
  "upload_url": null
}
```

**Validation:**
- `title`: required, 10–200 chars
- `price_cents`: required, min 100 (≥ $1.00)
- `sector`: required, must match enum list
- `product_type`: required, enum: `pdf_guide | checklist | template | swipe_file | mini_course`

---

#### `POST /api/v1/products/:id/assets`
Upload the product file. Multipart form upload.

**Request:** `multipart/form-data`
- `file`: the PDF or ZIP file
- `asset_type`: `main` (primary deliverable) | `bonus` | `preview`

**Response 200:**
```json
{
  "asset_id": "asset_abc123",
  "product_id": "prod_xyz789",
  "asset_type": "main",
  "filename": "solo-ai-stack.pdf",
  "size_bytes": 2048000,
  "r2_key": "products/prod_xyz789/main/solo-ai-stack.pdf"
}
```

**Constraints:**
- Max file size: 100MB
- Accepted types: `.pdf`, `.zip`, `.docx`, `.xlsx`
- File is stored in private R2 bucket (not publicly accessible)

---

#### `PUT /api/v1/products/:id`
Update any product metadata. Partial updates accepted (PATCH semantics).

**Request:**
```json
{
  "price_cents": 2700,
  "tags": ["AI tools", "productivity", "solopreneur"]
}
```

**Response 200:** Full product object with updated fields.

**Constraints:** Cannot update a published product's `sector` or `product_type`.

---

#### `POST /api/v1/products/:id/publish`
Publish the product. Triggers:
1. Validation (must have title, description, price, at least one `main` asset)
2. Landing page generation (LLM-generated hero copy, bullets, audience statement)
3. Stripe Product + Price object creation
4. Status → `live`

**Request:** (empty body)

**Response 200:**
```json
{
  "product_id": "prod_xyz789",
  "status": "live",
  "storefront_url": "https://skillforge.io/products/the-solo-ai-stack",
  "stripe_product_id": "prod_stripe_abc",
  "stripe_price_id": "price_stripe_xyz",
  "published_at": "2026-03-08T02:17:00Z"
}
```

**Error cases:**
- `400` — missing required fields or asset
- `409` — already published
- `422` — LLM landing page generation failed (agent should retry)

---

#### `GET /api/v1/products/:id`
Retrieve full product record.

**Response 200:** Full product object including status, URLs, asset list, Stripe IDs.

---

#### `GET /api/v1/products/:id/analytics`
Basic performance metrics.

**Response 200:**
```json
{
  "product_id": "prod_xyz789",
  "views": 142,
  "purchases": 7,
  "revenue_cents": 11900,
  "conversion_rate": 0.049,
  "period": "all_time"
}
```

---

#### `GET /api/v1/products`
List products with filters.

**Query params:** `sector`, `status`, `agent_id`, `limit` (default 20), `offset`

---

### 5.3 Error Response Format
All errors follow this structure:
```json
{
  "error": {
    "code": "MISSING_ASSET",
    "message": "Product must have at least one main asset before publishing",
    "field": "assets"
  }
}
```

Error codes: `UNAUTHORIZED`, `NOT_FOUND`, `VALIDATION_ERROR`, `MISSING_ASSET`, `ALREADY_PUBLISHED`, `LLM_GENERATION_FAILED`, `STRIPE_ERROR`, `UPLOAD_ERROR`

## 6. Non-Functional Requirements

### 6.1 Performance
- Product page load: < 2 seconds (LCP)
- API response time: < 500ms for all endpoints except `/publish` (< 10s — includes LLM call)
- File upload: supports up to 100MB, streaming upload to R2
- Checkout: Stripe hosted, not our responsibility

### 6.2 Reliability
- 99.5% uptime target for storefront
- API failures return clear error codes (never silent failures)
- File delivery: signed URLs with fallback re-generation on expiry

### 6.3 Security
- All API endpoints require Bearer token (except `/health`)
- Admin dashboard requires separate admin auth (Supabase Auth)
- R2 bucket is private — no direct public URLs for purchased files
- Stripe webhooks verified by signature
- No buyer PII stored beyond email + purchase record

### 6.4 Scalability
- Database designed to handle 100,000+ products from day one (proper indexing)
- R2 is effectively unlimited storage
- API should handle concurrent overnight runs (multiple SkillForge instances)

### 6.5 SEO
- Product pages server-side rendered (Next.js SSR/SSG)
- Auto-generated `meta title`, `meta description`, `og:image` per product
- Sector pages with proper h1/h2 hierarchy
- Sitemap auto-generated and updated on publish
